## Overview

Part 8 of this series tested whether training separate models for each event category improved on the unified XGBoost pipeline. The answer was no: the unified model won or tied in every segment during cross-validation, even when it appeared to lose on the raw test-set comparison. The core problem was data. With only ~30 features, segment-specific models did not have enough signal to generalize beyond the specific test window they were evaluated on.

Parts 10 and 11 changed that picture substantially. The feature set grew from ~30 to 58 features, adding lifecycle interaction terms (final-3-weeks window by segment), external signals (Spotify popularity, Wikipedia hype, team records), and inventory velocity. The Part 11 champion improved RMSE by 28% over Part 7.

With a richer feature set, the segmentation question was worth asking again. If the model now has segment-specific lifecycle signals baked in, maybe dedicated models trained only on their own category's data would learn to weight those signals more precisely.

This post documents that experiment. The conclusion mirrors Part 8: the global model wins. The reasons differ.

---

## Experimental Design

### The Baseline

The global baseline is the Part 11 champion: XGBoost classifier plus MLP regressor, trained on the full dataset across all segments, with threshold fixed at 0.60.

The test set is the final 3-week window: rows where `days_to_event <= 21`, covering the 389,230 event-week snapshots that represent the pricing-critical phase before each event. This is the same evaluation window used in Part 11.

The primary metric throughout is **active-events-only RMSE** — prediction error on rows where actual ticket sales occurred. Rows where no sales happened and the model correctly predicts zero contribute nothing to the error; this metric focuses on where the model is actually making substantive predictions.

### Segment Models

For each of the 7 focus buckets, I trained a fresh XGBoost classifier and XGBoost regressor using only that bucket's training data. Each stage received 30 Optuna trials using the same search space as the global model. Training ran on a Colab GPU (A100).

| Segment | Train Rows | Test Rows | Positive Test Rows |
|---------|------------|-----------|-------------------|
| Concerts | 1,905,356 | 129,770 | 37,901 |
| Broadway and Theater | 1,383,980 | 114,810 | 26,839 |
| Other | ~950K | 71,525 | 22,042 |
| Major Sports | 543,925 | 30,177 | 18,526 |
| Comedy | 463,932 | 35,079 | 8,458 |
| Minor Sports | ~95K | 6,827 | 3,191 |
| Festivals | 17,214 | 1,042 | 234 |

### Threshold Locking

This is the most important design choice, and it differs from Part 8.

In Part 8, the segment models could optimize their own thresholds, which gave them an implicit advantage: each could find the threshold that minimized RMSE on the test set. The global model used a fixed threshold from Part 7 (0.90 at the time). The comparison was not apples-to-apples.

For this experiment, the threshold is locked at 0.60 for every model, global and segment alike. This matches the threshold used in the Part 11 champion. The optimal per-segment threshold is still computed and reported, but it is not used in the primary comparison.

Significance was assessed with 1,000-resample bootstrap tests. A positive delta means the segment model is better (lower RMSE than global). The p-value is one-tailed: probability of observing a delta this large under the null that the two models are equivalent.

Festivals is the only segment where the dedicated model beats the global model by a statistically significant margin (p = 0.030). The confidence interval barely clears zero ([-0.04, +3.24]), so this result should be treated as marginal. Festivals also has the smallest test set at 1,042 rows and 234 positive examples, which makes bootstrap estimates noisier.

Comedy, Concerts, and Minor Sports show directional improvements ranging from 1% to 3.7%, but none cross the significance threshold. The bootstrap CIs for all three straddle zero.

Broadway and Major Sports both degrade, and their bootstrap distributions make it clear the regressions are not sampling noise.

## Broadway and Theater: Why Segmentation Hurts

Broadway regresses the most: 12.78 RMSE with the global model, 15.49 with a dedicated segment model (+21.2%). The p-value of 1.0 confirms this is not a sampling artifact; the bootstrap CI is [-3.75, -1.55], entirely below zero.

The segment model trains on 1.38M Broadway rows, which looks like adequate data. But Broadway has a specific structure that the global model already handles well. A large fraction of Broadway events have near-zero sales in any given week, and the classifier's job is to identify which weeks to pass to the regressor. The global classifier, trained on all seven segments, builds a more accurate representation of what distinguishes active from inactive event-weeks for Broadway, because it can anchor its Broadway-specific splits against the full distribution.

A Broadway-only classifier has never seen a concert or a sports event. It cannot calibrate Broadway's zero-sales rate relative to the broader market. This shows up in the threshold analysis: the Broadway segment model finds an optimal threshold of 0.10 (the search minimum), meaning it is essentially saying "always pass everything to the regressor." The global classifier at 0.60 is more selective and more accurate for this segment.

## Major Sports: More Features, Same Problem

Major Sports scores 39.25 global versus 41.73 segment (-6.3%). This runs counter to the original motivation: Part 11 added team records, win percentage, and win-loss streak specifically to address the Major Sports prediction gap. The expectation was that a dedicated model might learn to weight these signals more heavily.

Instead, the dedicated segment model is worse. The team record features improve predictions when trained globally, because the global model learns the relationship between those signals and sales across a larger and more varied sample. Training only on Major Sports events removes the contrast that makes those features useful.

The broader Major Sports problem also remains. The global RMSE for Major Sports is 39.25 tickets, far above every other segment. Team records and win streaks capture average-game context but not the game-specific factors that drive the highest-variance events: playoff rounds, nationally televised rivalry games, and championship implications. That variance ceiling was not moved by adding tabular signals about season-level performance, regardless of whether the model is global or segment-specific.

## The Threshold Collapse Pattern

The optimal threshold analysis reveals something worth noting. When threshold optimization is run on each segment's test set, five of seven segments find the optimal at 0.10, the minimum of the search range.

| Segment | Opt Threshold | Opt RMSE | Fixed RMSE (0.60) |
|---------|--------------|----------|------------------|
| Festivals | 0.80 | 13.66 | 13.66 |
| Minor Sports | 0.30 | 13.38 | 13.38 |
| Comedy | 0.10 | 13.67 | 13.69 |
| Concerts | 0.10 | 13.73 | 13.77 |
| Other | 0.10 | 32.54 | 32.56 |
| Major Sports | 0.10 | 41.58 | 41.73 |
| Broadway | 0.10 | 15.48 | 15.49 |

A threshold of 0.10 is nearly equivalent to "skip the classifier entirely and send every row to the regressor." When a model finds this optimal, it means the classifier is not discriminating well enough to justify filtering at 0.60. The global classifier at the same threshold would filter out a meaningful fraction of events; the segment classifier cannot reproduce that discrimination on its own segment's data.

The fact that Festivals finds an optimal threshold of 0.80 (and matching RMSE at both thresholds, 13.66) is the exception. The Festivals classifier is actually discriminative. This is plausibly why Festivals is the one segment where the dedicated model provides a real improvement: the segment model can learn a clean decision boundary for whether a festival event-week will sell, because festival demand patterns are distinct from other categories in ways the classifier captures well.

## Comparison to Part 8

Part 8 found no reliable benefit from segmentation using ~30 features and the Part 7 hyperparameter configuration. The conclusions here are consistent, but the failure mode is different.

In Part 8, the primary concern was generalization across time periods. The test-set improvement for Comedy (+16%) disappeared in cross-validation, suggesting the segment model was fitting to that specific test window rather than stable category patterns.

In this experiment, the test-set results are less encouraging to begin with. Only Festivals shows meaningful improvement, and the largest segments (Broadway, Concerts, Major Sports) are neutral to negative. The richer feature set did not create new opportunities for segmentation to help. If anything, it made the global model harder to beat: a 58-feature global model with lifecycle interactions and external signals already incorporates much of the segment-specific context that a dedicated model would be expected to capture.

The practical recommendation is unchanged: use the global model.

## Feature Importance by Segment

The feature importance patterns across segment models mirror what the Part 11 global analysis found. Market state features (`get_in_log_scaled`, `listings_active_log_scaled`, `sales_total_7d_log_scaled`) dominate in every segment. The lifecycle interaction terms (`final_3_weeks_major_sports`, `is_final_3_weeks`) appear prominently in sports segments. Spotify popularity ranks highest among external signals for Concert and Comedy models.

None of the segment models learn fundamentally different features from the global model. This mirrors the Part 8 finding: the Comedy segment model showed the lowest feature divergence from the unified model despite having the best test-set result. Segment models are not finding hidden signals. They are approximating the same function as the global model, but on less data.

## Conclusion

The global model trained on all data continues to outperform dedicated segment models. With 58 features that include segment-specific lifecycle interactions and external signals, the global model is already doing the work that segmentation was expected to add. Splitting training data by segment removes the cross-category context that makes the classifier and regressor accurate, without adding enough segment-specific signal to compensate.

The one partial exception is Festivals. Its segment model shows a significant improvement, and its classifier finds a non-trivial optimal threshold (0.80 versus the 0.10 floor seen elsewhere). The Festivals result is worth watching in a replication, but with only 1,042 test rows and a CI that barely excludes zero, it is not strong enough to justify maintaining a separate model in the pipeline.

The segmentation hypothesis has now been tested twice: with a basic feature set in Part 8, and with a rich one here. Both times, the unified model wins. The next experiments will focus on the high-variance segments directly, particularly Major Sports, rather than on architectural changes to the pipeline.
